# malloc の基礎

第 I 部では、アルゴリズムに入る前の土台を固める。

まず [malloc API の章](api-basics.md)で、`malloc`/`free`/`calloc`/`realloc` が
C 標準で**正確には**何を約束しているのか（そして何を約束していないのか）を確認する。
未定義動作の地雷原を地図にしておくことは、アロケータの実装を読むときにも役に立つ。

次に [OS とのインターフェースの章](os-interface.md)で、アロケータの「仕入れ先」である
カーネルとの境界——仮想メモリ、ページ、`brk` と `mmap`、`madvise`——を学ぶ。
malloc の設計判断の多くは、この境界のコスト構造から説明できる。

最後に[ヒープの中身を覗く章](inside-malloc.md)で、実際に動いている glibc malloc を
観察する。アドレスの規則性からチャンクヘッダを推理し、`strace` でシステムコールを
覗き、メモリが OS に返る瞬間を目撃する。第 II 部以降を読むための「現物感覚」を作る。
